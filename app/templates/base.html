Help & Settings Guide
What this does
Scans all folders under /media, finds video files not yet in H.265, and re-encodes them using your NVIDIA GPU (NVENC). The original file is replaced in-place. Plex and Jellyfin don't need to rescan. Already-processed files are cached and skipped on future scans.
CQ - Constant Quality
Controls quality vs. file size. Lower = better quality, larger file.

15-22  Near-lossless - file can be bigger than source
23-25  High quality, 20-40% smaller
26-28  Recommended - good quality, 40-60% smaller
29-32  Slight quality loss, 55-70% smaller
33+    Noticeable artifacts on film content

If you change CQ, files transcoded at a different CQ will be re-encoded on the next scan.
Preset
Controls how hard the encoder searches for compression. Unlike software codecs, NVENC is fixed-function hardware. The quality difference between fast and slow is small.

fast      Best throughput, minimal quality difference - recommended
medium  5-10% smaller files, 2x slower
slow     Marginal gain, 3x slower - not worth it
Parallel Jobs (Workers)
How many files encode simultaneously. All streams share the single NVENC unit.

1-2x  NVENC underutilized
3-4x  Recommended - ~80% NVENC load
5-6x  Good I/O overlap, high throughput
7-8x  VRAM pressure risk on 4 GB GPUs

CPU stays at ~10-20% regardless. All work is done on the GPU.
Output Container
The output container matches the source where possible. Plex and Jellyfin don't need to rescan because the file stays in the same folder with the same name.

.mp4 .m4v .mov → .mp4
.mkv → .mkv
.avi .wmv .flv .ts .mpg .mpeg .divx .webm → .mkv

The codec is always H.265 / HEVC. Audio and subtitle tracks are copied without re-encoding.
Reset DB vs Clean Jobs
Clean Jobs: deletes job history only. File cache is kept, so already-transcoded files will not be re-processed.

Reset DB: deletes job history and file cache. All files will be re-transcoded on the next scan as if they were never seen before.

Settings (CQ, preset, workers) are never affected by either action. They are stored separately in /data/settings.json.
Backup
Backs up the SQLite database (job history + file cache) to /data/backups/. Video files are not backed up.

Configure interval and retention via the Backup Settings button (download icon) in the header. Default: daily, keep last 7.
System Resources Bar (ETH)
The bar at the top shows live CPU, RAM, GPU, and Ethernet RX/TX. ETH reads from the host's /proc/net/dev via a read-only bind-mount. The container itself carries almost no traffic.

Required in docker-compose.yml:
- /proc/1/net:/host/proc/net:ro
Must be /proc/1/net, not /proc/net: the latter is a symlink that resolves to the container's own network namespace.
Display Theme (Auto / Light / Dark)
The theme button (sun/moon icon) in the header cycles through three modes:

Auto  - follows your device's system setting. If your Mac or phone switches between Dark and Light mode automatically (e.g. sunrise/sunset schedule), the dashboard follows along in real time.

Light  - forces light mode regardless of the system setting.

Dark  - forces dark mode regardless of the system setting.

The choice is saved in your browser. To reset to system-following behavior, click until the tooltip says "Auto".
